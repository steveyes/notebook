## conventions



Following 5 styles will be used in the rest of the documents since there are no **Constant Width** style in markdown.

- **Strong** - 

- *Emphasis* - *Warnings or important notes appear like this. *

- `code` - 

- > Quotes

- ```
  Code Fences
  ```



### usecases in table

| usecase                                                      | style                |      |
| ------------------------------------------------------------ | -------------------- | ---- |
| *Tips and tricks*                                            | Emphasis             |      |
| *email address*                                              | Emphasis             |      |
| *URL address*                                                | Emphasis             |      |
| *manual page citation in one line*                           | Emphasis             |      |
| *user input*                                                 | Emphasis             |      |
| **names of executeable programs**                            | Strong               |      |
| **pathnames**                                                | Strong               |      |
| **folder names**                                             | Strong               |      |
| **filenames**                                                | Strong               |      |
| **file extensions**                                          | Strong               |      |
| **database table names**                                     | Strong               |      |
| **environment variables**                                    | Strong               |      |
| **a new term**                                               | Strong               |      |
| **an important word**                                        | Strong               |      |
| ***dummy parameters that should be replaced with an actual value*** | Emphasis with Strong |      |
| `executeable program in one line`                            | Code                 |      |
| `command line in one line`                                   | Code                 |      |
| citations in multiple lines (*displayed in following paragraph*) | Quotes               |      |
| block of code (*displayed in following paragraph*)           | Code Fences          |      |



### usecase of **Quotes** - *citations in multiple lines*

> NAME
>        ps - report a snapshot of the current processes.
>
> SYNOPSIS
>        ps [options]



### usecase of **Code Fences** - *block of code* 

```
from django.contrib import admin
from .models import Post

admin.site.register(Post)
```



## notations

We leave it as understood that, when you enter a shell command, you press <kbd>Enter</kbd> at the end. <kbd>Enter</kbd> is labeled Return on some keyboards.

Characters called <kbd>Ctrl-X</kbd>, where *X* is any letter, are entered by holding down the <kbd>Ctrl</kbd> (or Ctl, or Control) key and then pressing that letter. 

Although we give the letter in uppercase, you can press the letter without the <kbd>Shift</kbd> key.

Other special characters are newline (which is the same as Ctrl-J),  Backspace (the same as Ctrl-H), Esc, Tab, and Del (sometimes labeled Delete or Rubout).
